2016-09-01
os
linux

目录说明

  • /bin : commands in this dir are all system installed user commands 系统的一些指令
  • /sbin: commands in this dir are all system installed super user commands 超级用户指令 系统管理命令，这里存放的是系统管理员使用的管理程序
  • /usr/bin: user commands for applications 后期安装的一些软件的运行脚本
  • /usr/sbin: super user commands for applications 超级用户的一些管理程序

BASH

  • 系统可用bash存放在/etc/shells文件中
  • 历史输入命令存在~/.bash_history文件中
  • 命令别名设置alias lm='ls -al'，此命令只在当前命令窗口有效，如需重启窗口后仍有效将此命令加入到~/.bashrc文件中即可
  • 删除别名：unalias [-a][别名] 参数 -a 表示删除全部的别名
  • 查询命令是否为Bash shell的内建命令：type，如type ls。type -a ls显示所有ls命令的信息（包括路径），并按执行顺序显示。所以type命令可以用来查看命令所在的路径。
  • 命令太长在一行放不开，可用\+回车续行。
    • Ctrl+u / Ctrl+k分别是从光标出向前或向后删除指令串
    • Ctrl+a / Ctrl+e分别是将光标移动到整个指令串的最前或最后面
  • mount重启有效，在/etc/fstab中添加

然后执行：sudo mount -a

变量

  • 定义变量：myname=VBird，等号两边不能有空格。变量值可以引用其他变量：var="lang is $LANG"，可使用转义符、双引号、单引号，使用单引号变量不能引用原值，会原样输出字面值。
  • 需额外命令提供值时，使用`命令`或\((命令)，如`version=\)(uname -r)，再echo $version`
  • 引用变量使用：$变量或${变量}
  • 定义的变量需在子程序中使用，用export使变量变成环境变量：export PATH
  • 删除变量：unset myname
  • 使用env命令查看环境变量，set命令查看所有变量及包括bash内置变量
  • 查看历史命令：history，!!执行上一个命令，!66执行第66个命令，!al执行最近以al为开头的命令
